# Playwright 资源消耗详细分析

## 📊 资源消耗对比

### 内存消耗

| 模式 | 单次运行 | 长时间运行 | 浏览器实例 |
|------|----------|------------|-----------|
| Headless | 200-500MB | 逐渐增加 | ~300MB |
| Headed | 500-1000MB | 显著增加 | ~300MB |
| Cookie方案 | 50-100MB | 稳定 | 无需 |

**问题**：
- ⚠️ 长时间运行会导致内存溢出
- ⚠️ 需要定期重启浏览器实例
- ⚠️ 并行运行可能消耗 90% 内存

---

### CPU 消耗

| 操作 | CPU 使用率 |
|------|-----------|
| 浏览器启动 | 40-60% |
| 页面加载 | 30-60% |
| JavaScript 执行 | 20-40% |
| 数据提取 | 5-10% |
| 并行运行（3个网站） | 接近 100% |

**对比 Cookie 方案**：
- Cookie 方案：5-10% CPU
- Playwright：30-60% CPU
- **差距 3-6 倍**

---

### 执行时间

| 操作 | Cookie 方案 | Playwright |
|------|-------------|-----------|
| 初始化 | 0秒 | 2-3秒 |
| 请求页面 | 0.5秒 | 3-5秒 |
| 解析数据 | 0.5秒 | 1-2秒 |
| **总计** | **1秒** | **6-10秒** |

**差距**：Playwright 慢 6-10 倍

---

### 磁盘消耗

| 项目 | Cookie 方案 | Playwright |
|------|-------------|-----------|
| 依赖安装 | 0 | 200-300MB |
| 浏览器缓存 | 0 | 100-200MB |
| 临时文件 | 0 | 50-100MB |
| **总计** | **0** | **350-600MB** |

---

## 🎯 适用场景分析

### ✅ 适合 Playwright

1. **反爬机制强的网站**
   - 需要执行 JavaScript
   - 需要模拟真人操作

2. **低频运行**
   - 每天 1-2 次
   - 不需要快速响应

3. **资源充足**
   - 内存 ≥ 8GB
   - CPU 性能良好

### ❌ 不适合 Playwright

1. **高频运行**
   - 每小时多次
   - 需要快速响应

2. **资源受限**
   - 内存 < 4GB
   - 低性能 CPU

3. **简单爬取**
   - 静态页面
   - API 可访问

## 🔧 优化建议

### 如果选择 Playwright

1. **使用 Headless 模式**
   - 减少 50% 内存消耗
   - 提高执行速度

2. **定期重启浏览器**
   - 防止内存溢出
   - 每次运行后关闭实例

3. **设置超时限制**
   - 防止卡死
   - 自动重试机制

4. **限制并发数**
   - 不要并行运行
   - 串行执行更稳定

## 🎯 推荐决策

**如果 Cookie 有效** → 用 Cookie 方案（最快最轻量）
**如果 Cookie 经常失效** → 用 Playwright（更稳定）
**如果有 RSS 源** → 用 RSSHub（最简单）
