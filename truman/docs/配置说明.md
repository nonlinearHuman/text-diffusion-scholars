# Truman 配置说明

## 环境变量配置

### Discord 配置
```bash
DISCORD_TOKEN=your_bot_token          # Discord Bot Token
DISCORD_CLIENT_ID=your_client_id      # Discord Client ID
DISCORD_GUILD_ID=your_guild_id        # Discord 服务器 ID
```

### 数据库配置
```bash
DATABASE_URL=postgresql://...         # PostgreSQL 连接字符串
```

### OpenClaw 配置
```bash
OPENCLAW_API_KEY=your_api_key         # OpenClaw API Key
```

### LLM 配置（上下文长度设置）

```bash
# LLM 模型选择
LLM_MODEL=glm-4-flash                 # 默认：glm-4-flash（性价比高）
# 可选：glm-4, glm-4-plus, gpt-4, claude-3-opus 等

# 上下文长度（控制 Agent 记忆窗口）
LLM_CONTEXT_LENGTH=4096               # 默认：4096 tokens
# 建议：
# - 2048: 简单场景，成本低
# - 4096: 平衡选择（推荐）
# - 8192: 复杂对话，保留更多历史
# - 128000: 使用 GLM-4-Long 等长上下文模型

# 最大输出 token 数
LLM_MAX_TOKENS=500                    # 默认：500
# 建议：
# - 100-200: 简短回复
# - 500: 正常对话（推荐）
# - 1000+: 详细回复

# 温度参数（控制创意性）
LLM_TEMPERATURE=0.7                   # 默认：0.7
# 建议：
# - 0.3-0.5: 更稳定、一致
# - 0.7: 平衡（推荐）
# - 0.9-1.0: 更有创意、多样
```

### 游戏配置

## 如何调整上下文长度

### 方法一：修改 .env 文件
```bash
# 编辑 .env 文件
nano /Users/gesong/.openclaw/workspace/truman/.env

# 修改这行
LLM_CONTEXT_LENGTH=8192  # 从 4096 改为 8192

# 重启 bot
pkill -f "node src/bot/index.js"
npm start
```

### 方法二：通过环境变量（临时测试）
```bash
LLM_CONTEXT_LENGTH=8192 npm start
```

### 方法三：Docker 部署时
```yaml
# docker-compose.yml
services:
  truman:
    environment:
      - LLM_CONTEXT_LENGTH=8192
```

## 上下文长度选择指南

### 🎯 场景一：简单互动（推荐起步）
```bash
LLM_CONTEXT_LENGTH=2048
LLM_MAX_TOKENS=200
```
- ✅ 成本低
- ✅ 响应快
- ⚠️ Agent 只能记住最近 10-20 条消息

### ⚖️ 场景二：平衡模式（默认）
```bash
LLM_CONTEXT_LENGTH=4096
LLM_MAX_TOKENS=500
```
- ✅ 性价比高
- ✅ Agent 能记住 20-40 条消息
- ✅ 适合大多数场景

### 🧠 场景三：深度对话
```bash
LLM_CONTEXT_LENGTH=8192
LLM_MAX_TOKENS=800
```
- ✅ Agent 能记住完整对话历史
- ✅ 适合复杂讨论
- ⚠️ 成本增加 2x

### 📚 场景四：长期记忆（高级）
```bash
LLM_MODEL=glm-4-long                  # 需要使用长上下文模型
LLM_CONTEXT_LENGTH=128000
LLM_MAX_TOKENS=1000
```
- ✅ Agent 能记住数周对话
- ✅ 适合长期社区运营
- ⚠️ 成本高

## 常见问题

### Q: 如何知道需要多少上下文长度？
A: 从 4096 开始，观察 Agent 行为。如果发现 Agent 经常"忘记"早期对话，增加长度。

### Q: 上下文长度越大越好吗？
A: 不一定。更长的上下文意味着：
- ✅ 更好的记忆
- ⚠️ 更高的成本
- ⚠️ 更慢的响应

### Q: 能否动态调整？
A: 可以。修改 .env 后重启 bot 即可，无需修改代码。
