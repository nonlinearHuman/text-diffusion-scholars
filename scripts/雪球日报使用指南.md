# 雪球日报使用指南

## 问题说明

雪球网页版有严格的反爬机制：
- 未登录访问会被拦截（显示"网络繁忙"或登录弹窗）
- 需要配置 Cookie 才能正常抓取

## 解决方案：配置 Cookie

### 方法一：浏览器获取 Cookie（推荐）

1. **打开 Chrome 浏览器**，访问 https://xueqiu.com
2. **登录雪球账号**（建议使用小号，避免主号风险）
3. **打开开发者工具**（F12 或 右键 → 检查）
4. **切换到 Network 标签**
5. **刷新页面**（F5）
6. **点击第一个请求**（通常是 `xueqiu.com`）
7. **在右侧 Headers 中找到 `Request Headers`**
8. **复制 `Cookie:` 后面的整串内容**

示例：
```
acw_tc=xxx; acw_sc__v2=xxx; xq_a_token=xxx; xq_r_token=xxx; ...
```

### 方法二：使用 EditThisCookie 插件

1. 安装 Chrome 插件：[EditThisCookie](https://chrome.google.com/webstore/detail/editthiscookie/)
2. 登录雪球后，点击插件图标
3. 点击 "Export" → "Header String"
4. 粘贴到配置文件

## 配置 Cookie

### 方式1：环境变量（推荐用于服务器）

```bash
# 临时设置（当前终端有效）
export XUEQIU_COOKIE='你的cookie值'

# 永久设置（添加到 ~/.zshrc 或 ~/.bashrc）
echo "export XUEQIU_COOKIE='你的cookie值'" >> ~/.zshrc
source ~/.zshrc
```

### 方式2：配置文件（推荐用于本地）

```bash
# 创建配置文件
echo '你的cookie值' > /Users/gesong/.openclaw/workspace/scripts/.xueqiu_cookie
```

## 注意事项

1. **Cookie 会过期**：通常 7-30 天，过期后需重新获取
2. **使用小号**：建议使用雪球小号，降低主号风险
3. **频率限制**：每天抓取一次，避免频繁访问被封

## 常见问题

### Q: Cookie 多久过期？
A: 通常 7-30 天，具体看雪球策略。过期后脚本会提示失败。

### Q: 为什么用小号？
A: 频繁自动化访问可能触发风控，使用小号可保护主号。

### Q: 脚本报错"网络繁忙"怎么办？
A: 检查 Cookie 是否正确配置，是否已过期。

### Q: 如何获取最新的段永平发言？
A: 脚本默认抓取昨天的发言，每天早上8点自动推送。
